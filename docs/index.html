Architectural Warning: Database Persistence

Understanding how data is written and saved within the application framework is crucial for a successful launch.

error
Local JSON File-Based Database

OrderOrbit records all session and transaction logs directly into the local file src/data/db.json. You must manage how this file behaves based on your platform architecture.

Vercel & Netlify (Serverless)

Serverless environments are stateless. Every time the function scales down to zero, or is redeployed, the local filesystem is wiped and reset. Consequently, any new orders or reviews written to db.json will disappear during serverless scaling.

Recommendation: Use Vercel or Netlify only for staging UI demonstrations. For production deployments on these platforms, modify src/data/db.ts to interface with a remote SQL/NoSQL cloud database (e.g. Supabase, MongoDB, or PostgreSQL).

Docker, Railway & Render (Containers)

Dockerized and container-based platforms maintain persistent states by mounting an external volume. By mapping a cloud volume to the directory `/app/src/data`, you ensure that the database file survives application restarts, updates, and rebuilds.

Make sure to check the volume mounting steps in the Docker, Railway, and Render guides.

Deploy on Vercel

Deploy your project natively to Vercel's Edge and Serverless infrastructure in just a few minutes.

  1. Push your project code to your Git account (GitHub, GitLab, or Bitbucket).
  2. Log in to the Vercel Console.
  3. Select Import Project and choose the OrderOrbit repository.
  4. Expand Environment Variables and enter all keys from the Checklist.
  5. Click Deploy. Vercel automatically runs the optimized build and assigns a domain name.
warning
Stateless Caveat

Remember that local writes to db.json are temporary. Placing orders or leaving feedback on a Vercel-hosted instance will reset regularly on function recycling.

Self-Hosted Docker Deployment

Deploy containerized workloads to a virtual private server (VPS) or cloud cluster using Docker volumes.

1. Build the Docker Image

Build your image in the project root directory where the Dockerfile and .dockerignore are placed:

Terminal Command 
docker build -t orderorbit .

2. Run Container with Volume Mounting

To persist data, mount a directory from your host operating system to the container's database folder: /app/src/data

Linux / macOS Docker Command 
docker run -d \
  -p 3000:3000 \
  --name orderorbit \
  -v /var/lib/orderorbit/data:/app/src/data \
  --env-file .env \
  orderorbit
Windows PowerShell Command 
docker run -d `
  -p 3000:3000 `
  --name orderorbit `
  -v C:\orderorbit\data:/app/src/data `
  --env-file .env `
  orderorbit

Deploy on Railway

Railway provides managed container Hosting with easy, point-and-click volume orchestration.

  1. Go to Railway.app and create a new project.
  2. Select Deploy from GitHub and connect your project repository.
  3. Under the service settings, navigate to Variables and click the Raw Editor. Paste all of your configuration parameters.
  4. Navigate to the **Settings** tab. Under **Volumes**, click **Add Volume**.
  5. Set the mount path to /app/src/data to guarantee that database edits are persistent.
  6. Save changes. Railway will build your container from the Dockerfile and start the service.

Deploy on Render

Deploying to Render as a Web Service utilizing Docker runtime and persistent disk mounts.

  1. Log in to the Render Dashboard.
  2. Select New > Web Service and choose your repository.
  3. Set the service runtime environment to Docker.
  4. Under Advanced Settings, configure a persistent disk volume:
    • Name: orderorbit-data
    • Mount Path: /app/src/data
    • Size: 1 GiB
  5. Add all configuration keys in the Environment Variables table.
  6. Create the Web Service to trigger the build and start serving requests.

Deploy on Netlify

Deploying Next.js applications using Netlify serverless functions and CDN distribution.

  1. Access Netlify and connect your GitHub profile.
  2. Click Import an existing project and choose the OrderOrbit repository.
  3. Netlify will auto-detect configurations:
    • Build command: npm run build
    • Publish directory: .next
  4. In Site configuration, add all environment variables.
  5. Click Deploy site to start the build.
warning
Stateless Caveat

Netlify utilizes ephemeral serverless functions. Local file alterations (such as updating db.json) do not persist across function instances.

Production Security Audit

Review this checklist before deploying to your production domain to protect transactions and user accounts.

  • Stripe Signatures: Always configure a valid STRIPE_WEBHOOK_SECRET in production. If the webhook secret matches the development default, signature validation is bypassed for easier testing.
  • JWT Strength: Use a securely generated, long key for JWT_SECRET. A 256-bit or 512-bit key ensures session cookies cannot be forged by students trying to hijack admin privileges.
  • Force HTTPS: Ensure your application URL is loaded over HTTPS (especially for Stripe and JWT authentication cookies).
  • Stripe API Modes: Check that you toggle off Stripe Test Mode (`sk_test_...`) and use Production Keys (`sk_live_...`) when taking actual payments from users.
